# Documentation

This directory contains detailed project documentation.

## Documents

### [requirements.md](./requirements.md)
Feature requirements, user stories, and acceptance criteria. Start here to understand what the application should do.

### [architecture.md](./architecture.md)
Technical architecture, design patterns, and technology choices. Read this to understand how the application is built.

### [database-setup.md](./database-setup.md)
Complete Supabase database setup including SQL migrations, RLS policies, and Google OAuth configuration.

### [results-visualization.md](./results-visualization.md)
Detailed specifications for 2D matrix and network graph visualizations, including scoring system and implementation examples.

### [dependencies.md](./dependencies.md)
Complete list of project dependencies, installation instructions, and library choices with rationale.

### [api-spec.md](./api-spec.md)
API endpoint specifications, request/response formats, and data contracts. Reference this when implementing or consuming the API.

## How to Use These Docs

1. **Planning a new feature?** → Start with `requirements.md`
2. **Need to understand the system?** → Read `architecture.md`
3. **Setting up the database?** → Follow `database-setup.md`
4. **Implementing visualizations?** → Reference `results-visualization.md`
5. **Installing dependencies?** → Check `dependencies.md`
6. **Building/consuming the API?** → Reference `api-spec.md`

Keep these documents updated as the project evolves!
